  What Needs Work

  1. API Design Mismatch (spec vs code)
  The spec says single-value ops should return StorageResult<T> directly, but
  the current code wraps everything in StorageItemStream. For example:
  - Spec wants: fn get(...) -> StorageResult<Option<V>>
  - Code has: fn get(...) -> StorageResult<StorageItemStream<'_, Option<V>>>

  This affects get, set, delete, exists, execute, execute_batch — all should
  return direct results per spec.

  2. Iron Law 4 Violations
  - Two #[allow(dead_code)] attributes on StorageProviderInner::D1 and ::R2
  variants (line 341-344 of storage_provider.rs)

  3. StorageProvider missing QueryStore dispatch
  StorageProvider implements KeyValueStore and RateLimiterStore dispatch, but
  there's no impl QueryStore for StorageProvider — so
  query/execute/execute_batch can't be called through the unified provider.

  4. Clippy blocked by upstream
  cargo clippy --package foundation_db -- -D warnings fails because
  foundation_core has an unused import (StreamIterator in valtron/task.rs:4).
  Not a foundation_db issue, but blocks verification.

  5. Phase 2 (foundation_auth integration) — Not started
  Tasks 11-16 from the spec (update foundation_auth to use foundation_db)
  haven't been touched.

  6. Spec metadata stale
  The feature.md says tasks: completed: 0, completion_percentage: 0% but most of
   Phase 1 is actually done.

  ---
  Recommended Next Steps (in order)

  1. Decide on API design: Should single-value ops return StorageResult<T>
  directly (per spec) or keep the current all-streams approach? This is a
  significant refactor that touches every trait, every backend, and every test.
  2. Fix #[allow(dead_code)] on D1/R2 — either remove the variants or add stub
  implementations.
  3. Add QueryStore impl for StorageProvider — dispatch to turso/libsql
  backends.
  4. Fix upstream clippy issue in foundation_core (the unused StreamIterator
  import).
  5. Phase 2: foundation_auth integration — the largest remaining chunk.
  6. Update spec metadata to reflect actual progress.

  cleanup or a significant refactor.
