# Changelog

## Unreleased

### Fixed

- **wrangler install failure** — mise was installing wrangler via its `npm:`
  shim (`"npm:wrangler" = "latest"` in `mise.toml`), which dropped it into a
  deeply nested `node_modules` tree under `~/.local/share/mise/installs/...`.
  Inside that layout, `sharp@0.34.5`'s `install/check.js` failed to resolve
  its prebuilt binary and fell back to a `node-gyp` source build, which in
  turn aborted with `sharp: Please add node-addon-api to your dependencies`.
  Moved wrangler (and `pnpm`) into the project's `devDependencies` so pnpm
  owns the install, and removed the broken `npm:wrangler` entry from
  `[tools]` in `mise.toml`. Added `_.path = ["./node_modules/.bin"]` under
  `[env]` so the existing `wrangler ...` invocations in the `cf:*` mise
  tasks resolve to the project-local binary.

- **`install:cloudflare` task** — fixed `pnmp:` typos and a stale pin to
  `wrangler@3.28.1`; the task now simply runs `pnpm install`.

- **`cf:start` / `cf:stop` leaked dev-server ports** — both tasks only ran
  `pkill -f "wrangler"`, which does not match the `workerd` child process
  that actually holds the dev-server port. A stale `workerd` from a prior
  run kept port `8789` bound and caused `wrangler dev` to fail with
  `Address already in use (127.0.0.1:8789)`. Both tasks now also
  `pkill -f "workerd"`.
