# Composable Templates

## Iron Law: Zero Warnings

> **All code must compile with zero warnings and pass all lints. No suppression. No exceptions.**
>
> - `cargo clippy -p foundation_deployment -- -D warnings -W clippy::pedantic` — zero warnings
> - `cargo doc -p foundation_deployment --no-deps` — zero rustdoc warnings
> - `cargo test -p foundation_deployment` — zero compilation warnings
> - **No `#[allow(...)]`, `#[expect(...)]`, or `#![allow(...)]` anywhere.** Fix the code, never suppress.

## Overview

Create composable project templates that generate:
1. **Deployment plans** (`.deployment-plan.yaml`) - The new declarative configuration format
2. **Legacy provider configs** (`wrangler.toml`, `service.yaml`, `template.yaml`) - For backward compatibility
3. **Complete project structure** - Build setup, source code, CI/CD, mise.toml

Templates are generated via `ewe_platform generate --lang rust --target cloudflare` or similar commands.

## Updated for Deployment Plans

**Key change:** Templates now generate `.deployment-plan.yaml` as the primary configuration, with optional legacy configs for backward compatibility.

```yaml
# .deployment-plan.yaml (generated by templates)
schema_version: "1.0"
provider: cloudflare
project: my-app
stage: production

service:
  name: my-worker
  main: src/lib.rs
  compatibility_date: "2026-01-01"
  
state_store:
  backend: d1
  database_name: my-app-state
```

## Requirements

### Template Matrix

| | Cloudflare | GCP Cloud Run | AWS Lambda |
|--|-----------|--------------|-----------|
| **Rust Worker** | `cf-rust-app` | - | - |
| **Rust WASM** | `cf-rust-wasm-app` | - | - |
| **Rust Container** | - | `gcp-rust-app` | `aws-rust-app` |
| **Rust Lambda** | - | - | `aws-rust-lambda` |

Each template generates:

```
{project-name}/
|-- Cargo.toml                  # (Rust templates)
|-- Dockerfile                  # (Container templates)
|-- .deployment-plan.yaml       # Primary config (new)
|-- wrangler.toml               # Legacy Cloudflare config (optional)
|-- service.yaml                # Legacy GCP config (optional)
|-- template.yaml               # Legacy AWS config (optional)
|-- mise.toml                   # Tooling and tasks
|-- README.md                   # Setup and usage
|-- .gitignore
|-- src/
|   +-- {entry-point}           # lib.rs / main.rs
+-- .github/
    +-- workflows/
        +-- deploy.yml          # CI/CD
```

### Template Variables

All templates use `ewe_temple` (MiniJinja) variable substitution:

| Variable | Description | Example |
|----------|-------------|---------|
| `{{project_name}}` | Project name | `my-api` |
| `{{project_name_snake}}` | Snake case name | `my_api` |
| `{{author}}` | Author name | `darkvoid` |
| `{{provider}}` | Target provider | `cloudflare`, `gcp`, `aws` |
| `{{stage}}` | Deployment stage | `production`, `staging` |
| `{{region}}` | Deployment region | `us-central1` |

### Template Structure

```rust
// src/template/mod.rs

use ewe_temple::TemplateEngine;

/// Template engine for generating projects.
pub struct TemplateEngine {
    engine: TemplateEngine,
}

impl TemplateEngine {
    /// Generate a new project from a template.
    pub fn generate(
        &self,
        template_name: &str,
        project_name: &str,
        provider: ProviderType,
        stage: StageName,
        output_dir: &Path,
    ) -> Result<(), TemplateError>;
}

/// Available templates.
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum TemplateKind {
    // Cloudflare
    CfRustApp,
    CfRustWasmApp,
    // GCP
    GcpRustApp,
    GcpRustJob,
    // AWS
    AwsRustApp,
    AwsRustLambda,
}
```

### Generated Deployment Plan

```yaml
# .deployment-plan.yaml.j2 (Cloudflare template)
schema_version: "1.0"
provider: cloudflare
project: {{ project_name }}
stage: {{ stage | default('production') }}

service:
  name: {{ project_name }}
  main: src/lib.rs
  compatibility_date: "2026-01-01"
  
  environment:
    RUST_LOG: info
    
state_store:
  backend: d1
  database_name: {{ project_name }}-state

build:
  command: npm run build
  context: .
```
