# Zero Warnings Workspace - Eliminate All Rust Lints

## Overview

This specification covers the workspace-wide effort to eliminate **all** Rust
compiler warnings, clippy lints, and documentation warnings. The goal is
**zero warnings** with **zero suppression** except where explicitly justified.

### Golden Rules

1. **Never suppress** — Fix the underlying code, don't silence the warning.
2. **llama.cpp FFI exception** — `infrastructure/llama-bindings/` and
   `infrastructure/llama-cpp/` may use `#[allow(...)]` for FFI boundary
   code where the C API dictates patterns (e.g., `missing_safety_doc`,
   `not_unsafe_ptr_arg_deref`). This is the **only** blanket exception.
3. **Generated code rule** — For code generated by tools
   (`bin/platform/gen_resources/`, `crates/html-macro/`, `crates/template-macro/`):
   - **First:** Fix the generator, regenerate, verify clean.
   - **Last resort:** If no alternative exists, suppression at the **top of the
     generated file only** (module-level `#![allow(...)]`), documented with a
     `// GENERATED:` comment pointing to the generator source.
   - **Never edit generated files directly** — dead code, unused imports, or
     other warnings in generated output mean the **generator** is wrong. Fix
     the generator template, then regenerate.
4. **No dead code (hand-written only)** — Remove unused code from hand-written
   files; don't `#[allow(dead_code)]` it. Generated files follow rule 3.
5. **No `todo!` / `unreachable!` stubs** — Either implement or remove.

## Implementation Strategy

### Per-Crate Workflow

1. **Capture** — `cargo clippy --package <crate> --all-targets 2>&1 | tee /tmp/<crate>.clippy`
2. **Categorize** — Group warnings by lint type and file
3. **Mechanical** — Run `cargo clippy --fix --package <crate> --all-targets` for auto-fixes
4. **Manual** — Fix remaining warnings by hand (doc comments, signatures, etc.)
5. **Generator sweep** — For generated code, find the generator source, fix it, regenerate
6. **Verify** — `cargo check/clippy/test --package <crate>` all pass
7. **Commit** — One commit per crate with message `[cleanup] <crate>: zero warnings`

### Generated Code Locations

| Generator | Output Location | Fix Strategy |
|-----------|-----------------|--------------|
| `bin/platform/src/gen_resources/` | `bin/platform/src/gen_resources/*.rs` | Fix generator in `gen_resources/types.rs`, regenerate |
| `crates/html-macro/` | `crates/html-macro/test/**/*.rs` | Fix macro, re-run `cargo test` to regenerate |
| `crates/template-macro/` | `crates/template-macro/test/**/*.rs` | Fix macro, re-run tests |
| `mime_guess` (build script) | `target/debug/build/*/out/mime_types_generated.rs` | Not in git — ignore (target/ is gitignored) |
| `infrastructure/llama-bindings/` | Bindings via bindgen | FFI exception — `#[allow]` at module level OK |

### Suppression Policy (Exceptions)

**Allowed (with documentation):**

| Location | Reason | Suppression Scope |
|----------|--------|-------------------|
| `infrastructure/llama-bindings/src/lib.rs` | FFI boundary, C API patterns | Module-level `#![allow(...)]` |
| `infrastructure/llama-cpp/src/*.rs` | Safe wrapper over FFI | Per-item only if C API forces it |
| Generated files (if unavoidable) | No generator fix possible | Top-of-file `#![allow(...)]` with comment |

**Never allowed:**

| Pattern | Action |
|---------|--------|
| `#[allow(clippy::doc_markdown)]` on hand-written code | Fix: add backticks |
| `#[allow(dead_code)]` on unused functions | Fix: remove the function |
| `#[allow(clippy::cast_possible_truncation)]` | Fix: use `try_into` or prove bounds |
| `#[allow(clippy::expect_used)]` | Fix: use `?` or match |
| `#[allow(clippy::todo)]` | Fix: implement or remove |
| `#[allow(clippy::missing_errors_doc)]` | Fix: add `# Errors` section |
| `#[allow(clippy::needless_pass_by_value)]` | Fix: change to `&T` or `impl Into<T>` |

## Risks and Mitigations

| Risk | Mitigation |
|------|------------|
| Signature changes break downstream | Fix crate-by-crate in dependency order; verify dependents after each |
| Generated code changes break tests | Run full test suite after regenerating; fix generator first |
| `todo!()` stubs have no clear implementation | Flag for user decision; don't implement blindly |
| `expect()` calls hiding real error cases | Audit each; convert to `?` where errors can propagate |
