## Key Design Principles

### 1. Provider Abstraction

The `pi-ai` layer defines a unified interface for AI models across all providers. Providers register themselves via `registerApiProvider()` with `stream()` and `streamSimple()` functions. The `streamSimple()` function provides a simplified interface with unified options (e.g., `reasoning` for thinking level), while `stream()` exposes provider-specific options.

### 2. Event-Driven Agent Loop

The agent loop emits `AgentEvent` values for every lifecycle event: `agent_start`, `turn_start`, `turn_end`, `message_delta`, `message_done`, `tool_execution_start`, `tool_execution_end`, `agent_end`. This enables multiple UI modes (TUI, print, JSON, RPC) to subscribe to the same event stream.

### 3. Model-as-Descriptor

Models are described by a `Model<TApi>` interface containing: `id`, `name`, `api`, `provider`, `baseUrl`, `reasoning`, `input`, `cost`, `contextWindow`, `maxTokens`, `compat`. This descriptor drives all provider routing decisions.

### 4. Auto-Generated Model Catalog

The model catalog (`models.generated.ts`, ~14,830 lines) is auto-generated by `scripts/generate-models.ts`. This keeps hundreds of models across all providers synchronized without manual maintenance.

## Data Flow

```
User Input -> AgentSession.prompt() -> Agent.prompt()
  -> agentLoop() -> streamAssistantResponse() -> Provider.stream()
  -> AssistantMessageEvent stream -> tool calls detected
  -> executeToolCalls() -> results pushed to context
  -> repeat until no tool calls -> agent_end event
  -> AgentSession processes events -> persist, compact, retry check
```
